04 — In-site editing

Click to edit. Anywhere.

After you log in once with FCADMIN, every public page on the site reveals dashed boxes around editable elements. Click → type → autosaves after 2 seconds.

Go to any page

e.g. /kids

Toggle edit mode on

Look for the small "Edit" pill in the corner of the screen (usually top-right or bottom-right). Click it.

Dashed boxes appear

Warm-toned outlines around every editable text/photo. The badge in each box shows: who owns this slot + slot name.

Click a box, type

The box becomes editable. Just type. No save button. No publish button.

Stop typing for 2 seconds

You'll see "saved · just now" appear in the corner. That's autosave. Live on the site immediately.

Verify (optional)

Reload the page (Cmd+Shift+R). Your text is still there → confirmed live.

Common mistakes to avoid

  • Don't paste from Word with formatting. Hidden formatting can break the layout. Paste as plain text (Cmd+Shift+V) or paste into TextEdit first.
  • Don't use ALL CAPS for emphasis. The site has its own typography rules. Caps will look shouty.
  • Don't write "click here" or "see below". Visitors may arrive deep-linked. Write so each paragraph stands alone.
  • Don't overthink it. First pass: write something honest in your voice. Second pass next week: refine. Done is better than perfect.
The "saved" indicator

After you stop typing, watch for one of: Saved · just now (good) Saving… (in progress) Save failed — retry (network hiccup, click retry). If you see no indicator at all → assume your edit didn't save. Reload and check.

05 — Photos

The campus pool curator

Each campus has a photo pool — a folder of images that the pastor (or their delegate) has uploaded. Then you curate which photo goes into which slot on the campus's page (the hero, the secondary, the leaders portrait, etc.). One pool, multiple destinations.

Where to find the photo curator for a campus

/intake/admin/[campus-slug]/photos

Examples:

  • /intake/admin/paradise/photos
  • /intake/admin/adelaide-city/photos

The two halves of the curator page

HalfWhat it shows
Top — The poolEvery photo the pastor uploaded. Thumbnails. Caption. Assign button.
Bottom — The slotsHero photo, Secondary, Leaders portrait, Mosaic 1–4, Service photo. Each shows what's currently assigned.

The Futures photo brief — what makes a "good" photo

✓ Good✗ Bad
Real moments, not posed Stock photography (visitors can sniff stock from a mile away)
Faces in focus, eyes visible Empty auditoriums with stage lights
Outside the building when possible Heavily filtered / Instagram-style
Diverse — different ages, ethnicities, contexts Selfies (wrong scale, wrong context)
Captioned with location/year if known Group shots where everyone's squinting at the sun
Don't be shy about removing things

A campus page with 4 great photos beats a campus page with 12 mediocre ones. If a pool has nothing strong, ask the pastor for more. If they can't deliver, use what's least-bad in the hero slot and leave the mosaic slots empty.

07 — When you're stuck

Don't fix it. Capture it. Text Ashley.

The first rule: you don't fix anything technical yourself. You spot the problem, capture the right details, and text Ashley. The system has been engineered to be repaired by Ashley in 5–30 minutes once flagged.

The 30-second "is this real?" check

  1. Hard reload — Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
  2. Try incognito/private window — rules out browser extensions
  3. Try a different network — phone hotspot
  4. Try a different device — your phone if you were on laptop

If the problem persists across all of those, it's real. Escalate.

The most common 5 problems

ProblemFirst tryIf still broken
Can't log in Confirm FCADMIN all caps, no spaces. Hard reload. Text Ashley with browser + last-working time
An edit won't save Refresh — your text might've actually saved. Try a different slot. If only this slot fails: text Ashley with slot name. If all slots fail: call immediately.
A pastor's link doesn't work Re-check the CSV. Re-issue from /intake/admin/seed. Text Ashley if seed page misbehaves
No dashed boxes appearing Hard reload. Confirm logged in (/intake/admin works?). Try a different page. Text Ashley
Milo not responding Try a different question. Refresh. Text Ashley — likely a quick API quota fix

Escalation chain

SeverityExamplesResponse time
🟢 MinorTypo in label, confusing tooltipWithin 24h
🟡 MediumOne link broken, one slot failingWithin 4h
🔴 MajorLogin down, all saves failing, offensive content liveImmediate. By phone if no text reply in 15 min.

The "what to send Ashley" checklist

  1. What URL (exact link)
  2. What you did (clicked X, typed Y)
  3. What you expected
  4. What actually happened
  5. What you tried (hard reload, incognito, etc.)
  6. Browser + device (Chrome on Mac, etc.)
"Hey — login failing at /intake/admin/login. Typing FCADMIN all caps, expecting /intake/admin. Form says wrong password. Tried hard reload, incognito, different network. Chrome on Mac. Started 10 min ago."

That message gets fixed in 5 minutes.
